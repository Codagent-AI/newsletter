### Validating the Validator

> - SEGUE: Validation starts at the local level, and review is an important part of that. So how well do local code review agents in Agent Validator actually work?
> - CLAIM: The surprising part from the evals is not "use the strongest model everywhere." 
> - SOURCE: `/Users/paul/codagent/agent-validator/docs/eval-report-2026-04-05.md`
> - EVAL SETUP: The fixture had 56 planted issues across three languages: TypeScript, Python, and Go.
>   - Code quality: 24 issues. Bugs, maintainability problems, performance traps, type-safety problems.
>   - Security: 17 issues. SQL injection, secrets in code, path traversal, SSRF, auth bypasses, weak randomness.
>   - Error handling: 15 issues. Swallowed errors, masked failures, false success responses, ignored cleanup failures.
> - REVIEW SETUP: I tested the three built-in local code review agents as separate passes first.
>   - Code Quality reviewed only the code-quality fixture, where the security and error-handling bugs were fixed.
>   - Security reviewed only the security fixture, where the code-quality and error-handling bugs were fixed.
>   - Error Handling reviewed only the error-handling fixture, where the other bugs were fixed.
> - COMBINED SETUP: Then I tested broader prompts.
>   - Combined all-areas prompt: one prompt covering code quality, security, and error handling against all 56 issues.
>   - Security + error-handling combined: one prompt against the 32 non-code-quality issues, with code-quality bugs fixed.
>   - This is the important comparison: three small reviews versus one or two broader reviews, not just one model versus another model.
> - APRIL BASELINE: The best production-value setup was a two-agent Copilot hybrid:
>   - Code review agent 1: GitHub Copilot CLI + `claude-sonnet-4.6`, low effort, specialized code-quality prompt.
>   - Code review agent 2: GitHub Copilot CLI + `gpt-5.3-codex`, medium effort, combined security + error-handling prompt.
>   - Result: about 0.74 weighted recall across 56 issues, about 0.85 precision, about 178s total.
>   - Why it worked: Sonnet dominated code-quality review, while GPT-5.3 did surprisingly well when security and error handling were combined into one prompt.
>   - Cost/quality angle from the report: roughly 40-50% of the cost of running three separate Sonnet reviews, with only a small recall tradeoff versus the best absolute recall setup.
> - SURPRISE: Copilot was the best adapter in these tests.
>   - Claude Code CLI + Sonnet timed out at 300s.
>   - The same Sonnet model through Copilot CLI finished reliably and produced the best code-quality recall.
>   - Through Copilot CLI, GPT-5.3 was not the best standalone code review agent, but became strong when used for the right combined prompt.
>   - For Copilot CLI + GPT-5.3, the security + error-handling combined prompt was the sweet spot: 0.79 recall in one about-73s pass, better than running security and error handling separately.
> - SURPRISE: Codex CLI worked best with one broad prompt.
>   - For Codex CLI + GPT-5.3, the combined all-areas prompt beat the three separate specialized prompts on recall and sharply improved precision.
>   - That single Codex CLI pass covered code quality, security, and error handling across all 56 issues.
>   - Several issues that were never found by separate code review agents became findable in combined mode.
> - CODEX STORY: Codex is also good, but only with the right configuration.
>   - Recommended fallback when Copilot is unavailable: Codex CLI + `gpt-5.3-codex`, medium effort, combined all-areas single prompt.
>   - Result: 0.69 mean recall, 0.96 precision, about 82s.
>   - Important caveat: Codex GPT-5.3 performed much worse on separate specialized prompts, so the combined prompt is mandatory for this story.
> - SOURCE: `/Users/paul/codagent/agent-validator/docs/eval-report-2026-05-02.md`
> - MAY UPDATE: The May evals mostly did *not* change the recommendation.

## Agent Runner

> PAUL:
>
> At the top, I described my full workflow: task loop, Validator, assumptions review, /simplify, another Validator run, archive, PR, CI, Agent Reviews. And I said two things that sound a little too convenient: it takes no time from me, and none of the steps get skipped.
>
> The key is Agent Runner, which I introduced last time as the missing coordination layer in the outer harness.
>
> Claude and Codex have task lists. They can follow instructions. They can even keep a checklist. But complex workflows with nested loops are a different problem.
>
> The usual answer is to put the workflow inside the agent's context and shout: NEVER skip steps, execute ALL steps in order, do NOT stop early. That helps, but it is still prompt-following. The workflow lives inside the context window, which makes it a suggestion, not a guarantee.
>
> Agent Runner flips that around. Instead of the agent calling my code when it happens to remember, my code tells the agent what to do next.
>
> It is a deterministic workflow layer outside the agent's context window. The model does what it's good at: reasoning, judgment, adaptation. The workflow layer does what it's good at: sequencing, validation, enforcement.
>
> So I don't have to remember to run the task loop, then the validator loop inside each task, then the assumptions review, then /simplify, then Validator again, then archive the OpenSpec change, then create the PR, then wait for CI and Agent Reviews, then fix what they find.
>
> Agent Runner does that for me. It runs those steps automatically and *autonomously*, in sequence. It keeps going while I'm doing something else, like planning the next change.
>
> That's why it takes no time from me.
>
> And that's why none of the steps get skipped. The workflow is the checklist. The task loop, the validator retry loop, the assumptions review, /simplify, the second Validator run, the archive, the PR, CI, Agent Reviews, and fixes all have a place in the sequence. PR creation is part of the workflow. Waiting for CI is part of the workflow. Addressing review comments is part of the workflow.
>
> So when I come back to the task, I can trust that it's already been verified for correctness and quality. I can instead focus on what I do best, steering the ship (link to https://codagent.beehiiv.com/p/the-middle-agentic-path).
